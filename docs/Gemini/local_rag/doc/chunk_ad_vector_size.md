Отвечаю на ваши вопросы по поводу работы RAG-системы и предлагаю готовое решение для обновления:

### Меняется ли размер чанка в зависимости от размера вектора?

**Нет, размер чанка не зависит от размера вектора.** Размер вектора (размерность) — это длина математического массива (координат), который возвращает модель (например, 768 для `nomic-embed-text` и 384 для `all-minilm`). Она никак не связана с ограничением на количество принимаемого текста (контекстным окном).

Размер чанка в коде жестко зафиксирован на безопасном значении `max_chars=150` символов (что составляет примерно 40–100 токенов в зависимости от языка и наличия формул). Это сделано намеренно для обеспечения **стабильности работы**:

1. Разные языковые модели эмбеддингов имеют жесткий лимит физического размера батча (обычно 256 или 512 токенов).
2. Формулы, знаки препинания и русские символы требуют гораздо больше токенов на один символ, чем стандартный английский текст.
3. Ограничение в 150 символов гарантирует, что ни один кусок текста (даже самая сложная математическая формула) не вызовет ошибку переполнения контекста Ollama (`input is too large to process`), сохраняя стабильность индексации на слабом процессоре.

2. Запустите скрипт на вашем компьютере:
```bash
python3 patch_app.py
3. Перезапустите контейнеры, чтобы изменения применились в Docker-окружении:
