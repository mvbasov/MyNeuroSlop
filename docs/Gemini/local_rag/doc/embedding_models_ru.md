# Выбор эмбеддинг-модели для русского языка на слабом ПК (8GB RAM / 4 CPU)

Эмбеддинг-модель `all-minilm` быстрая и легкая, но её векторное пространство практически не содержит семантики русского языка. Ниже представлены три лучшие альтернативы, оптимизированные под ваши системные требования.

## Сравнение подходящих моделей

Мы разделим кандидатов на два типа: **работающие внутри Ollama** (проще настроить, но делят память с LLM) и **работающие прямо в Python** через библиотеку `sentence-transformers` (требует изменения кода, но высвобождает гигабайты памяти Ollama).

| **Модель** | **Способ запуска** | **Размер на диске** | **Размер вектора (dim)** | **Качество (RU)** | **Нагрузка на RAM/CPU** | 
 | ----- | ----- | ----- | ----- | ----- | ----- | 
| **`nomic-embed-text`** | Ollama | \~274 МБ | **768** | Хорошее | Низкая (очень быстрая) | 
| **`bge-m3`** | Ollama | \~1.2 ГБ | **1024** | Отличное (SOTA) | Средняя/Высокая | 
| **`rubert-tiny2`** | Python (HuggingFace) | **\~116 МБ** | **312** | Отличный баланс | Экстремально низкая | 

### Вариант 1. `nomic-embed-text` (Через Ollama) — Лучший компромисс

Это современная легковесная модель от Nomic AI. Она отлично поддерживает мультиязычность, включая русский язык, и при этом работает невероятно быстро даже на CPU.

* **Плюсы:** Нативно поддерживается в Ollama, весит всего \~274 МБ, имеет длинный контекст (до 8192 токенов, хотя для CPU лучше использовать ваши стандартные 256-512).
* **Минусы:** Чуть уступает BGE-M3 в сложных математических терминах.
* **Размер вектора:** `768`

**Как скачать в Ollama:**
```bash
docker exec -it ollama ollama pull nomic-embed-text
```

### Вариант 2. `bge-m3` (Через Ollama) — Максимальное качество

Лучшая открытая мультиязычная модель от Beijing Academy of Artificial Intelligence (BAAI). Если вам нужен максимально точный поиск по техническим текстам на русском языке.

* **Плюсы:** Невероятная точность, отлично понимает формулы, код, русско-английский суржик и специфическую терминологию.
* **Минусы:** Требует около 1.2 ГБ RAM. На 4 CPU процесс индексации крупных файлов может быть медленным.
* **Размер вектора:** `1024`

**Как скачать в Ollama:**
```bash
docker exec -it ollama ollama pull bge-m3
```

### Вариант 3. `rubert-tiny2` (Внутри Python) — Архитектурный лайфхак для 8 ГБ RAM

Если Ollama на вашей машине часто падает с ошибкой `500` (Out of Memory) при одновременной работе LLM (Qwen) и эмбеддинга, лучшим решением будет **вынести эмбеддинги из Ollama напрямую в ваш Python-контейнер `rag-app`**.

Мы можем использовать специализированную русскую модель `cointegrated/rubert-tiny2` через стандартную Python библиотеку `sentence-transformers`.

* **Плюсы:**
  1. **Освобождает RAM:** Ollama больше не тратит память на эмбеддинги и полностью отдает свои ресурсы под генерацию ответов (LLM).
  2. **Скорость:** Модель весит смешные 116 МБ и на CPU кодирует текст мгновенно.
  3. **Локальность:** Работает без сетевых запросов к Ollama внутри приложения.
* **Размер вектора:** `312`

## Пошаговый план перехода на новую модель (на примере `nomic-embed-text`)

Поскольку у вас уже настроена система очистки кэша и инициализации Qdrant, изменение размерности вектора требует обязательного сброса базы данных.

### Шаг 1. Остановите контейнеры
```bash
docker compose down
```

### Шаг 2. Очистите старую базу Qdrant
Старая база была создана под размерность `all-minilm` (384). Прямая вставка векторов новой размерности вызовет ошибку `400 Bad Request`.
```bash
sudo rm -rf qdrant_storage
```

### Шаг 3. Измените конфигурацию в вашем `app.py`
Откройте файл `app.py` и отредактируйте глобальные переменные и функцию инициализации коллекции:

1. **Измените имя модели и размерность:**
   ```python
   EMBED_MODEL = "nomic-embed-text"  # Или "bge-m3"
   VECTOR_SIZE = 768                # 768 для nomic, 1024 для bge-m3
   ```

2. **Убедитесь, что в функции создания коллекции Qdrant используется правильный размер:**
   При инициализации коллекции (обычно метод `create_collection` или `recreate_collection`) значение `size` должно быть привязано к вашей новой переменной:
   ```python
   client.recreate_collection(
       collection_name="rag_documents",
       vectors_config=VectorParams(size=VECTOR_SIZE, distance=Distance.COSINE),
   )
   ```

### Шаг 4. Обновите `docker-compose.yaml` (модуль автоматического скачивания)
В вашем сервисе `model-puller` (если вы его используете для прогрева контейнера) замените команду скачивания старой модели на новую:
```yaml
# Пример строки в docker-compose для model-puller:
command: >
  /bin/sh -c "
  echo 'Pulling embedding model...';
  curl -s -X POST http://ollama:11434/api/pull -d '{\"name\": \"nomic-embed-text\"}' &&
  echo 'Embedding model cached!';
  "
```

### Шаг 5. Запустите систему заново
```bash
docker compose up -d --build
```

После запуска скрипт `app.py` обнаружит отсутствие папки `qdrant_storage`, создаст чистую коллекцию под размерность нового вектора и запустит переиндексацию ваших файлов на русском языке.

## Резюме: что выбрать?

1. Если хотите **просто переключить имя в коде** и не нагружать систему: выберите **`nomic-embed-text`**. Это самый стабильный и сбалансированный вариант для вашего ПК.
2. Если критически важно **максимальное качество поиска по сложным русским терминам** и вы готовы пожертвовать парой секунд при индексации: выберите **`bge-m3`**.
3. Если система продолжает выдавать **ошибки 500 / зависать**: напишите мне, и мы перепишем логику получения эмбеддингов в `app.py` на автономную работу с **`rubert-tiny2`** без участия Ollama. Это навсегда решит проблему нехватки памяти.
